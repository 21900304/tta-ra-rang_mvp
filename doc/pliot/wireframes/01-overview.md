# 01. WireFrame 설계 개요

## 📊 IA 분석 요약

### 서비스 특성
- **유형**: B2B EdTech + 헬스케어 융합 (특수교육 체육 플랫폼)
- **핵심 가치 제안**:
  1. 설치 없이 즉시 사용 가능한 PWA 기반 AI 체육 수업 도우미
  2. 실시간 다중 포즈 추적으로 6명 학생 동시 관리
  3. 게이미피케이션으로 특수학생 동기부여 극대화

### MVP 범위 (P0)
- 실시간 다중 포즈 추적 (최대 6명)
- 캐릭터 1:1 매칭 및 시각화
- 클래스 기반 순차 운영 카운팅
- 기본 AI 피드백 (자세/격려)
- 교사 제어판
- 캘리브레이션
- 수업 결과 요약

### 기술적 제약
- PWA (Desktop Web), Chrome/Edge 90+
- MoveNet AI 모델, 20-30 FPS 목표
- WCAG AA, CUD (Color Universal Design)
- 듀얼 스크린 (PC + TV)

---

## 🎯 WireFrame 설계 원칙 3가지

### 1. Dual-Screen First
교사용(제어/모니터링)과 학생용(피드백/게이미피케이션) 화면을 완전히 분리하여 각 사용자의 멘탈 모델에 최적화합니다.

### 2. Zero-Depth Navigation
특수학생의 인지 부하를 최소화하기 위해 모든 핵심 기능을 1-2클릭 내에서 접근 가능하도록 설계합니다.

### 3. High Contrast & Large Touch Targets
지적장애 학생을 고려하여 색상+모양을 병행 사용하고, 44px 이상의 터치 영역을 확보합니다.

## 🔧 전문 프레임워크 참고

### Nielsen Norman Group 원칙
1. **Visibility of System Status**: 사용자가 현재 위치와 상태를 항상 알 수 있어야 함
2. **User Control and Freedom**: 실수를 되돌릴 수 있는 명확한 경로 제공
3. **Consistency and Standards**: 동일한 정보/기능은 동일한 위치와 표현 유지
4. **Recognition Rather than Recall**: 사용자가 기억하기보다 인식할 수 있도록 설계
5. **Flexibility and Efficiency**: 초보자도 쉽게, 숙련자는 빠르게 사용 가능

### WCAG 2.1 접근성 기준
- **대비비**: 텍스트 4.5:1 이상 (AA 기준), UI 요소 3:1 이상
- **키보드 접근성**: 모든 기능을 키보드로 조작 가능
- **Focus 상태**: 명확한 포커스 표시 (2px 이상 outline)
- **다중 모드**: 색상+모양+텍스트 병행 사용 (색맹 대응)

### 7±2 법칙 (Miller's Law)
- **메뉴 항목**: Top-level은 5-7개 이하로 제한
- **화면당 정보**: 주요 요소는 3-5개 이하로 집중
- **선택지**: 한 번에 제시하는 옵션은 7개 이하

## 📝 주요 사용자 여정 (Journey)

### 여정 1: 첫 수업 시작 (신규 사용자)

**목표**: 첫 사용 후 5분 내 수업 시작 완료

```
1. 메인 화면 진입
   → <Dashboard />

2. 클래스 선택
   → 드롭다운 클릭
   → 클래스 카드 선택

3. "수업 시작하기" 버튼 클릭
   → handleStartClass() 호출
   → 라우팅: / → /calibration

4. 캘리브레이션
   → <Calibration />
   → 학생 배치 및 손 들기 (자동 인식)
   → 매칭 확인

5. "확인 완료" 버튼 클릭
   → 라우팅: /calibration → /session/teacher
   → <TeacherView />

총 클릭 수: 4회
예상 소요 시간: 3분
검증 결과: ✅ IA 목표 "5분 내 수업 시작" 충족
```

### 여정 2: 수업 중 모니터링

**목표**: 실시간으로 학생들의 진행 상황 확인 및 제어

```
1. 수업 진행 화면
   → 실시간 6인 그리드 확인
   
2. 학생별 진행률 확인
   → 각 학생 카드 내 프로그레스 바
   
3. 피드백 로그 확인
   → 우측 사이드바 피드백 섹션
   
4. 일시정지/재개
   → 하단 제어 바 "일시정지" 버튼
   
5. 다음 운영으로 스킵
   → "다음 운영" 버튼 클릭

core 기능 접근: 1-2클릭 내
검증 결과: ✅ 핵심 기능 즉시 접근 가능
```

### 여정 3: 수업 종료 및 결과 확인

**목표**: 수업 종료 후 결과 저장 및 새 수업 준비

```
1. "수업 끝내기" 버튼 클릭
   → 확인 모달 표시
   
2. 결과 화면 자동 표시
   → 라우팅: /session/teacher → /results
   → <Results />
   
3. 학생별 성과 확인
   → 스크롤로 학생 카드 확인
   
4. 결과 저장
   → "결과 저장" 버튼 클릭
   
5. 새 수업 시작
   → "새 수업 시작" 버튼 클릭
   → 라우팅: /results → /

총 클릭 수: 3회
검증 결과: ✅ 결과 저장 및 새 수업 시작 원활
```

## 🚨 중요 원칙 (절대 위반 금지)

1. **사용자 중심**: 기술/비즈니스 로직이 아닌 사용자 멘탈 모델 우선
2. **단순성**: 복잡함은 제품이 아닌 구현에서 해결 (2레벨 이상 깊이 없음)
3. **일관성**: 같은 것은 같게, 다른 것은 다르게 (교사-학생 역할 명확 분리)
4. **접근성**: 모든 사용자를 위한 설계 (CUD 원칙, WCAG AA)
5. **확장성**: 오늘의 선택이 내일을 막지 않도록 (클래스 관리에 Post-MVP 통합)
