# 01. 설계 개요 및 원칙

## 📊 서비스 분석

### 서비스 프로파일
- **유형**: B2B EdTech + 헬스케어 (특수교육 체육)
- **핵심 가치 제안**:
  1. 설치 없이 즉시 사용 가능한 PWA 기반 AI 체육 수업 도우미
  2. 실시간 다중 포즈 추적으로 6명 학생 동시 관리
  3. 게이미피케이션으로 특수학생 동기부여 극대화

### MVP 범위 (P0)
| 기능 | 설명 |
|------|------|
| 실시간 다중 포즈 추적 | 최대 6명 동시 추적 |
| 캐릭터 1:1 매칭 | 학생-캐릭터 시각화 |
| 클래스 기반 순차 운영 | 스쿼트 → 푸쉬업 → 버피 |
| 기본 AI 피드백 | 자세 교정, 격려 메시지 |
| 교사 제어판 | 수업 시작/일시정지/종료 |
| 캘리브레이션 | 학생-캐릭터 매칭 |
| 수업 결과 요약 | 성과 데이터 제공 |

### 기술적 제약
- **플랫폼**: PWA (Desktop Web), Chrome/Edge 90+
- **성능**: MoveNet AI, 20-30 FPS 목표
- **접근성**: WCAG AA, CUD (Color Universal Design)
- **하드웨어**: 듀얼 스크린 (PC + TV)

## 🎯 설계 원칙 3가지

### 1. Dual-Screen First
교사용(제어/모니터링)과 학생용(피드백/게이미피케이션) 화면을 완전히 분리
- 교사: 좌측 사이드바 + 상단 헤더 + 중앙 그리드
- 학생: 전체화면 캐릭터 뷰 + 최소 텍스트

### 2. Zero-Depth Navigation
특수학생 인지 부하 최소화
- 모든 핵심 기능 1-2클릭 내 접근
- 메뉴 깊이 최대 2레벨 유지
- 단계별 진행 표시 (Progress Bar)

### 3. High Contrast & Large Touch Targets
지적장애 학생 고려한 접근성 설계
- 색상 + 모양 + 텍스트 병행 (CUD 원칙)
- 최소 44x44px 터치 영역
- 고대비 모드 지원

## 🔧 적용된 디자인 프레임워크

### Nielsen Norman Group 10가지 휴리스틱

1. **시스템 상태 가시성**: 현재 위치와 진행 상태를 항상 표시
2. **시스템과 실세계의 일치**: 사용자의 언어와 개념 사용
3. **사용자 제어와 자유**: 실수를 되돌릴 수 있는 경로 제공
4. **일관성과 표준**: 동일한 정보/기능은 동일한 위치와 표현
5. **오류 방지**: 오류 발생 전에 방지하는 설계
6. **회상보다 인식**: 기억하기보다 인식할 수 있도록
7. **유연성과 효율성**: 초보자도 쉽고, 숙련자는 빠르게
8. **미적 최소주의**: 불필요한 정보 제거
9. **오류 인식/진단/복구**: 명확한 오류 메시지와 해결 방법
10. **도움말과 문서**: 필요 시 쉽게 접근 가능

### WCAG 2.1 접근성 기준

| 기준 | 요구사항 | 적용 |
|------|----------|------|
| 대비비 | 텍스트 4.5:1 이상 | ✅ 모든 텍스트 적용 |
| 키보드 접근성 | 모든 기능 키보드 조작 | ✅ Tab 순서 논리적 |
| Focus 상태 | 2px 이상 outline | ✅ 명확한 포커스 표시 |
| 다중 모드 | 색상+모양+텍스트 | ✅ CUD 원칙 적용 |

### 7±2 법칙 (Miller's Law)

- **메뉴 항목**: Top-level 4개 (대시보드, 수업, 클래스, 설정)
- **화면당 정보**: 주요 요소 3-5개 집중
- **선택지**: 한 번에 7개 이하 옵션 제공

## 📝 용어 체계 (Labeling System)

### 원칙
1. **명확성 > 창의성**: "수업 시작" > "Let's Move!"
2. **간결성**: 2-4단어 이내
3. **일관성**: 유사 기능은 동일 용어
4. **사용자 언어**: 교사는 전문 용어 가능, 학생용은 초등 수준

### 핵심 용어

| 개념 | 한글 | 영문 | 사용 위치 |
|------|------|------|-----------|
| 수업 시작 | 수업 시작 | Start Class | 메인 버튼 |
| 수업 종료 | 수업 끝내기 | End Class | 제어 버튼 |
| 클래스 | 클래스 | Class | 네비게이션 |
| 운영 | 운영 | Exercise | 수업 화면 |
| 캐릭터 | 캐릭터 | Character | 전체 |
| 캘리브레이션 | 자리 맞추기 | Calibration | 안내 |
| 횟수 | 횟수 | Count | 수업 화면 |
| 완료 | 완료 | Complete | 피드백 |
| 피드백 | 피드백 | Feedback | 수업 화면 |
| 설정 | 설정 | Settings | 네비게이션 |
| 대시보드 | 메인 화면 | Dashboard | 네비게이션 |

### 피해야 할 용어
- 애매한 용어: "항목", "것", "기타"
- 전문 용어 (학생용): "캘리브레이션", "임계값", "인식"
- 플랫폼 의존적: "클릭", "마우스"
- 부정적 표현: "실패", "틀림" → "더 해보기", "연습하기"

## 🚨 중요 원칙 (절대 위반 금지)

1. **사용자 중심**: 기술/비즈니스 로직이 아닌 사용자 멘탈 모델 우선
2. **단순성**: 복잡함은 제품이 아닌 구현에서 해결
3. **일관성**: 같은 것은 같게, 다른 것은 다르게
4. **접근성**: 모든 사용자를 위한 설계
5. **확장성**: 오늘의 선택이 내일을 막지 않도록
