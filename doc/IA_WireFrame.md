### 1.2 User Flow

```
[랜딩 페이지]
    ↓
[수업 시작하기 클릭]
    ↓
[클스 선택] → [학생 수 설정] → [디스플레이 설정]
    ↓
[수업 시작]
    ↓
[Play Mode - 교사용 제어판]
    ↓ (동시에)
[Student Display - 학생용 화면] ← Broadcast Channel 연동
    ↓
[수업 종료] ← [종료 버튼] 또는 [모든 운�� 완료]
    ↓
[완료 Dialog] → [홈으로] 또는 [다시 시작]
```

```
sessionStorage
├── classConfig: ClassConfig

    ├── id

    ├── students[]

    ├── isPaused
    └── currentExercise
```

## 2. WireFrame Structure Guide

### 2.0 레이아웃 기본 구조

**모든 페이지 공통 레이아웃:**
```
┌─────────────────────────────────────────────────────────────┐ ← Header
│  [좌측 액션]  [중앙: 로고/타이틀]  [우측 액션]              │
├─────────────────────────────────────────────────────────────┤ ← Main Content
│                                                             │
│                    [페이지별 콘텐츠 영역]                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**구조 원칙:**
- Header는 페이지 상단에 고정
- Main Content는 Header 아래에 위치
- 콘텐츠는 중앙 정렬 또는 좌측 정렬
- 모바일에서는 세로 스택, 데스크톱에서는 그리드 활용

├─────────────────────────────────────────────────────────────┤
│  VALUE SECTION                                              │

│  │              │  │              │  │              │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
│  [3개의 가치 제안 카드]                                     │

**섹션 구성:**
- **Header**: 좌측에 로고, 우측에 정보 버튼
- **Hero**: 중앙 정렬, 타이틀 → 설명 → 캐릭터 → CTA 버튼 순
- **Value Props**: 3열 그리드, 각 카드는 아이콘 + 타이틀
- **System Requirements**: 알림 스타일 박스

## 3. Key Interactions

### 3.1 Navigation Flow
1. **Home → Setup**: CTA 버튼 클릭
2. **Setup Steps**: 이전/다음 버튼 또는 Step Indicator 클릭
3. **Setup → Play**: 수업 시작하기 버튼 클릭
4. **Play ↔ Display**: BroadcastChannel을 통한 실시간 동기화
5. **End Dialog**: 홈으로 또는 다시 시작 선택

### 3.2 Play Mode Controls
- **시작/일시정지**: 단일 버튼 토글
- **다음 운동**: 다음 운동으로 이동
- **리셋**: 현재 운동 초기화 (확인 대화상자)
- **종료**: 수업 완전 종료 (확인 대화상자)

### 3.3 Student Progress Tracking
- **AI Pose Detection**: Canvas 위에 노란 점으로 표시 (Mock)
- **실시간 업데이트**: 1초마다 시뮬레이션
- **피드백 메시지**: 운동 유형별 동적 생성
- **완료 축하**: Student Display에서 애니메이션 효과

## 4. 상태 및 예외 처리

### 4.1 로딩 상태
- **초기 로딩**: 스피너 중앙 배치
- **데이터 로딩**: 스켈레톤 UI
- **카메라 연결**: "카메라 연결 중..." 메시지

### 4.2 에러 상태
- **카메라 권한 거부**: 안내 메시지 + 설정 안내
- **BroadcastChannel 오류**: 연결 재시도 버튼
- **데이터 없음**: Empty State 표시

### 4.3 빈 상태 (Empty State)
- **Display 페이지**: "수업 준비 중..." 메시지
- **학생 목록**: "학생 정보 없음" 안내
- **울동 완료**: 축하 애니메이션
