- Design Essence Item 확인용
    
    ```jsx
    그럼 반대로 첨부한 레퍼런스(Web UI, App UI 등)의 구조적인 요소만 추출하고 싶으면 어떤 항목을 기반으로 요청하고, 어떻게 프롬프팅 해야해?
    ```

- DeepSeek Prompting(deeper thinking)
    
    ```jsx
    # 🎯 요청 개요
    나는 지금부터 첨부한 UI 레퍼런스의 구조적 디자인 요소만 추출하고 싶어.
    컬러, 폰트, 이미지 스타일, 무드, 그래픽 장식 등 시각적 스타일은 완전히 배제하고,
    레이아웃, 정보 구조, 컴포넌트 배치, 흐름, 시스템만 분석해줘.
    
    # 📋 분석 항목 (8가지)
    아래 8가지 항목을 기준으로 레퍼런스를 분석해줘.
    각 항목은 구체적인 수치, 위치, 관계, 방식을 중심으로 서술해줘.
    감성적 표현이나 스타일적 평가는 하지 말아줘.
    
    1. **전체 레이아웃 구조**
       - 화면을 어떻게 분할하는가?
       - 주요 영역의 구성과 배치는?
    
    2. **그리드 시스템**
       - 컬럼 수, 마진, 거터의 관계
       - 정렬의 기준과 그리드 준수/이탈
    
    3. **정보 계층 구조**
       - 중요 요소의 위치와 시선 흐름
       - 정보 우선순위에 따른 배치 차등화
    
    4. **컴포넌트 배치와 구성**
       - 네비게이션, 카드, 폼, 오버레이 등의 위치와 배치 방식
       - 콘텐츠 표현 단위 (리스트, 그리드, 캐러셀 등)
    
    5. **인터페이스 흐름**
       - 화면 전환 방식
       - 상태 변화와 피드백의 위치
    
    6. **밀도와 여백의 활용**
       - 정보 밀도
       - 간격의 일관성과 여백의 기능
    
    7. **정렬과 그룹핑**
       - 정렬 기준
       - 정보 묶음의 단위와 구분 방식
    
    8. **반응형/적응형 구조의 단서**
       - 고정/유동 여부
       - 확장/축소 시 예상되는 구조 변화
    
    # 📌 최종 출력 형식 요청
    - 위 8가지 항목을 각각 3~5줄의 서술형 문장으로 정리해줘.
    - 수치나 비율은 구체적으로 언급해줘. (예: 12컬럼, 마진 24px, 거터 16px 등)
    - 위치 표현은 모호하지 않게 해줘. (예: "왼쪽 상단" 대신 "헤더 영역 우측", "우측 하단 플로팅 버튼" 등)
    - 마지막에 전체 구조 시스템을 한 단락으로 요약해줘.
    - 절대 "세련된", "모던한", "깔끔한" 등의 스타일 평가어를 쓰지 마.
    ```
    
- Gemini Prompting(deeper thinking)
    
    ```jsx
    너는 15년 경력의 시니어 UX 아키텍트이자 프로덕트 디자이너야. 너의 임무는 첨부된 레퍼런스에서 스타일적 요소(컬러, 폰트, 질감, 이미지의 분위기)를 완벽히 제거하고, 인터페이스의 **'기능적 골격(Skeleton)'과 '정보 설계(IA)'**만 역설계(Reverse Engineering)하는 것이야.
    
    이미지의 색상과 심미적 스타일을 무시하고, 다음의 4가지 항목을 기반으로 UI 구조 보고서를 작성해줘:
    
    1. **그리드 및 배치 논리 (Grid & Spacing):** 화면을 구성하는 수직/수평 그리드 구조를 분석해줘. 요소들 사이의 간격(Padding/Margin) 규칙과 정렬 방식(Alignment)이 어떻게 정보를 그룹화하고 있는지 설명해줘.
    
    2. **정보 계층 구조 (Information Hierarchy):** 화면 내에서 가장 강조된 요소(Focal Point)부터 보조 정보까지의 위계를 분석해줘. 사용자가 정보를 인지하는 순서가 어떻게 설계되었는지 '와이어프레임' 관점에서 서술해줘.
    
    3. **내비게이션 및 기능 블록 (Functional Navigation):** 주요 메뉴의 위치와 유형(Top, Side, Bottom 등)을 정의하고, 각 기능 단위(예: 히어로 섹션, 리스트 컨테이너, 카드 시스템)가 어떻게 나뉘어 있는지 분석해줘.
    
    4. **컴포넌트 조립 방식 (Atomic Structure):** UI를 구성하는 개별 컴포넌트들이 어떻게 결합되어 있는지 분석해줘. (예: '아이콘 + 텍스트 + 배경'이 하나의 '리스트 아이템' 분자를 구성하는 방식 등)
    
    [Constraints]
    - 색상(예: "파란색 버튼"), 폰트(예: "Sans-serif"), 특정 이미지 묘사는 절대 포함하지 마.
    - 대신 "주요 액션 버튼(Primary CTA)", "계층적 목록(Nested List)", "콘텐츠 컨테이너"와 같은 UX/UI 구조 용어를 사용해.
    - 결과물은 실제 개발을 위한 와이어프레임 설계도처럼 논리적이고 건조하게 작성해줘.
    ```
    
- Grok Prompting(deeper thinking)
    
    ```jsx
    너는 12년차 수석 UI/UX 디자이너이자 인터페이스 구조 전문가야.
    
    첨부된 레퍼런스 이미지(Web UI 또는 App UI)의 **순수한 구조적 요소**만 분석해줘.
    
    **절대 분석하지 말아야 할 것** (완전히 제외):
    - 색상, 컬러 팔레트
    - 폰트, 타이포그래피 스타일
    - 미학적 느낌, 분위기, 스타일, 형태 언어, 텍스처
    - 선의 질감, 그림자 스타일 등 시각적 디테일
    
    오직 아래 8가지 구조적 항목에만 집중해서 분석해줘:
    
    1. 전체 레이아웃 아키텍처 (Overall Layout Architecture)
    2. 정보 계층 구조 (Information Hierarchy)
    3. 네비게이션 시스템 (Navigation System)
    4. 콘텐츠 조직 및 그리드 시스템 (Content Organization & Grid System)
    5. 컴포넌트 배치 패턴 (Component Placement & Relationships)
    6. 여백과 공간 전략 (Spacing & White Space Strategy)
    7. 화면 상태 및 플로우 구조 (Screen States & User Flow)
    8. 모듈러/반복 패턴 (Modular & Reusable Patterns)
    
    **출력 형식**:
    - 각 항목을 bullet point로 작성
    - 각 항목당 2~4문장으로 구체적으로 설명 (가능하면 실제 레이아웃 비율, 배치 논리, 사용자 흐름 관점에서 분석)
    - 마지막에 
      • **한 줄 핵심 구조 요약** (한 문장)
      • **대표 구조 키워드** 6~8개
    
    이 레퍼런스를 새로운 UI를 설계할 때 구조 참고용으로 활용할 수 있도록, 실용적이고 명확하게 분석해줘.
    ```
    
- Claude Prompting(Sonnet)
    
    ```jsx
    첨부한 UI 레퍼런스의 구조적 요소만 분석해줘.
    
    **분석 제외 항목:**
    - 컬러 스킴
    - 폰트/타이포그래피 스타일
    - 시각적 장식 요소 (아이콘 스타일, 일러스트, 질감 등)
    
    **추출해야 할 구조 항목:**
    
    1. **레이아웃 구조**
       - 화면 분할 방식과 영역 구성
       - 그리드 시스템 (몇 컬럼, 정렬 방식)
       - 주요 섹션의 배치 패턴
    
    2. **정보 계층**
       - 콘텐츠의 우선순위 구조
       - 1차/2차/3차 정보의 배치 원칙
       - 정보 전달의 시각적 흐름
    
    3. **네비게이션 패턴**
       - 메뉴/탭의 구조와 위치
       - 주요 이동 경로 설계
       - Breadcrumb이나 페이지네이션 방식
    
    4. **컴포넌트 구성**
       - 사용된 주요 UI 컴포넌트 목록
       - 카드/섹션/모듈의 조직 방식
       - 반복 패턴의 단위 구조
    
    5. **콘텐츠 배치 원칙**
       - 텍스트/이미지/버튼의 배열 규칙
       - 요소 간 간격과 정렬 체계
       - 그룹핑과 시각적 구획 방법
    
    6. **상호작용 구조**
       - CTA 버튼의 위치와 배치 전략
       - 폼 구조와 입력 흐름
       - 스크롤 영역과 고정 요소
    
    **출력 형식:**
    - 각 항목을 구체적으로 분석하되, 시각적 스타일은 언급하지 말 것
    - 와이어프레임으로 표현할 수 있는 수준의 구조적 정보만 추출
    - 이 구조를 다른 프로젝트에 적용할 수 있는 설계 원칙으로 정리
    ```
